Stitch 工作流

先生成界面，再调整审美方向

  1. 打开 Google Stitch。
  2. 输入页面类型、目标用户、内容模块和视觉风格。
  3. 选择最接近目标的版本，继续修改颜色、版式和模块。
  4. 将设计结果转化为页面结构和代码任务。

创建仓库

保存代码并管理版本

  • 打开 github.com/new。
  • Repository name 填 stitch 或自定义项目名。
  • 选择 Public，便于使用免费 GitHub Pages。
  • 不要勾选 README / .gitignore / License。
Terminal
cd ./your-project
git init
git add -A
git commit -m "init website"
git branch -M main
git remote add origin https://github.com/your-name/your-repo.git
git push -u origin main
1. 创建仓库获得远程代码地址。
2. 推送代码把本地文件同步到 GitHub。
3. 后续更新修改、提交、推送即可同步。
入门推荐

GitHub Pages

适合静态作品集、课程页、个人主页。路径清晰，和 GitHub 仓库直接关联。

适用：HTML / CSS / JS
域名友好

Cloudflare Pages

适合绑定自定义域名、连接 GitHub 自动部署。可用于 Astro、静态站和轻量前端项目。

适用：个人网站 / 品牌站
前端项目

Vercel

适合 Next.js、React 等项目，预览部署体验好。

适用：现代前端框架
替代选择

Netlify

适合表单、重定向、静态站点部署，配置简单。

适用：营销页 / 文档站
GitHub Pages 设置
  1. 进入仓库 Settings
  2. 选择左侧 Pages
  3. Source 选 Deploy from a branch
  4. Branch 选 main / root
  5. 保存后获得访问地址
